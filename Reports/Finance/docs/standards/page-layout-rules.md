# Page Layout Rules

## Purpose

Use these rules to preserve a disciplined management-report layout across the report.

## Design Direction

- follow the active benchmark shell
- preserve a quiet, executive, CFO-facing tone
- prefer structure and clarity over decorative complexity

## Layout Hierarchy

The default page hierarchy should be:
- filters first
- KPIs second
- analysis visuals third
- detailed tables or statements last

## Repeated Patterns

- treat KPI rows as reusable systems, not one-off cards
- treat slicer rails as reusable systems, not page-specific experiments
- standardize spacing, labels, and control treatment across similar pages

## Slicer Rules

- the preferred left rail uses separate text labels above compact dropdowns
- avoid heavy slicer-level styling overrides unless necessary
- preserve order and spacing consistency across pages that share the same control set

## Card Rules

- keep titles business-facing
- avoid duplicate-caption behavior where both helper captions and visual titles appear
- prioritize legibility over dramatic styling

## Chart And Container Rules

- keep pages airy and uncrowded
- preserve strong white container structure from the benchmark shell
- avoid adding visuals that duplicate an existing business story without a clear reason

## Layout Change Rule

If a page needs structural layout change, update repeated patterns intentionally rather than screenshot-matching one isolated page.
