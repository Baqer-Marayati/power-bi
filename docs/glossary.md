# Glossary

## Purpose

This glossary keeps recurring project language consistent across documentation and future work sessions.

## Terms

### `PBIP`
Power BI project format used as the editable source of truth in this repository.

### `PBIX`
Single-file Power BI file that may be useful for review or transfer, but is not the editable master for this project.

### `TMDL`
Tabular Model Definition Language files used to define the semantic model.

### `Semantic Model`
The model layer behind the report, including tables, measures, relationships, and formatting logic.

### `Benchmark Shell`
The reference report structure and visual language taken from the active benchmark, currently `Design Benchmarks/Sample 2`.

### `Compatibility Logic`
Temporary model or report logic used to keep visuals working when direct SAP-backed truth is not yet available.

### `SAP-Backed Truth`
Logic that comes from the intended real source domain rather than a temporary compatibility workaround.

### `Project Memory`
The retained markdown knowledge base in this repository that stores current truth, decisions, risks, and next steps.

### `Repo-Facing Docs`
The stable orientation layer at the root of the repository, especially `README.md` and the `docs/` folder.
